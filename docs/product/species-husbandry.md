# Species Husbandry

Fluxpoint stores husbandry as collection-scoped guide data on a species definition, then lets individual aquarium inventory items carry sparse local overrides.

## Core Model

- `SpeciesHusbandryGuide` belongs to one collection and one `SpeciesDefinition`.
- `SpeciesHusbandryOverride` belongs to one collection and one `AquariumItem`.
- Species definitions may be global seed records, but guides and overrides are always collection-scoped.
- Empty override forms delete the override instead of keeping blank rows.

Guide statuses:

- `LOCAL`: normal editable collection guide.
- `LINKED`: guide resolves live from another same-collection species definition.
- `AI_DRAFT`: draft content generated by Eddy and waiting for review.
- `REVIEWED`: content has been reviewed by the collection.

## Field Registry

The source of truth for husbandry fields is:

`src/domains/husbandry/husbandry-fields.ts`

Species types:

- `FRESHWATER_FISH`
- `MARINE_FISH`
- `PLANT`
- `INVERTEBRATE`
- `CORAL`
- `OTHER`

Each type has its own sections and fields. UI forms, inline editing, normalization, override merging, and badges all use this registry.

## Linking And Forking

Linked guides are for species that should share the same living husbandry source. Fluxpoint blocks:

- self-links
- link cycles
- links across collections
- inline edits on linked guides

Use **Fork linked guide** when a species needs its own copy. Forking copies the resolved source fields, summary, difficulty, and type into the local guide, then removes the live link.

## Aquarium Item Overrides

Aquarium detail pages show effective husbandry for stocked species. Effective husbandry is:

`resolved guide fields + non-empty item override fields`

Overrides are intended for tank-local realities such as a specific group, quarantine notes, compatibility notes, diet changes, or a different lighting/flow condition.

## AI Magic Fill

`/api/ai/species-husbandry-fill` returns a draft only. It does not save guide data.

The route:

- requires an authenticated collection user
- only accepts species definitions visible to that collection
- normalizes output through the husbandry field registry
- records the attempt in `AiRequestLog`
- falls back to mock draft content when the configured AI provider is unavailable

Users must copy or manually enter AI draft content into the guide form before it becomes durable husbandry data.

## Eddy Context

Aquarium AI context includes compact husbandry summaries from effective item husbandry. This keeps Eddy aware of species-specific needs without sending the full guide payload.

## Integrity Check

Run:

```bash
npm run check:husbandry-integrity
```

The check fails on cross-collection guide mismatches, invalid links, cycles, linked guides with local edits, and stale overrides whose species no longer matches the aquarium item.
